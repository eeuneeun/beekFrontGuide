1. HTML 코드 작성 규칙

HTML 코드를 작성하는 Mark up 을 진행 할 때 다음과 같은 기본 규칙을 준수합니다.

  1. 1 W3C Validation 을 통과 할 수 있는 구조로 작성 할 것

    HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation 을 통과해야 합니다.
    단, HTML5 DTD 선언 시 다음 오류 내용은 허용 할 수 있습니다.

    • iframe 의 frameborder, marginwidth, marginheight, scrolling 애트리뷰트

  2. 2 tag 작성시에는 영문 소문자만을 사용 할 것

    DTD 문서 명세를 제외한 모든 요소와 attribute는 소문자로 작성해야 합니다.

  3. 3 tag 내부의 attribute 와 class, id 등의 표기는 쌍따옴표("") 를 사용 할 것

    attribute, class, id 를 비롯한 tag 내부의 모든 속성 값은 쌍따옴표만을 사용 합니다.

  4. 4 tag 내부에 들어가는 특수문자는 웹 표준을 준수하여 코드로 작성 할 것

    tag 내부의 모든 특수문자는 HTML 5의 Character references 에 기재되어 있는 코드 값으로 표기 합니다.

    • HTML5 Character Rreferences Link

  5. 5 inline 선언은 가급적 지양 할 것

    tag 내부에 inline 방식으로 사용하는 css 속성 및 스크립트는 지양해야 합니다.

  6. 6 들여쓰기 규칙은 다음의 규칙을 준수 할 것

    들여쓰기를 하면 코드의 가독성이 높아지고 전체 HTML 구조를 쉽게 파악 할 수 있어 전체적인 코드의 재생산성을 높여줍니다.
    BEEK 에서는 마크업의 중첩이 깊어질 때마다 자식 요소는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 규정합니다.
    문서 내에서는 반드시 탭을 이용하여 들여쓰기를 하며, 탭을 대신하여 공백으로 띄어 들여쓰지 않도록 유의합니다.

  7. 6 빈 줄을 사용하려면 다음의 규칙을 준수 할 것

    의미 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 만드는 것을 허용합니다.
    다만, 빈 줄은 1줄을 초과하지 않도록 유의합니다.

2. CSS 코드 작성 규칙

CSS 코드를 작성하는 Mark up 을 진행 할 때 다음과 같은 기본 규칙을 준수합니다.

  1. 1 W3C Validation 을 통과 할 수 있는 방식으로만 작성 할 것

    CSS는 사용 가능한 Hack과 CSS3 속성을 제외하고 W3C Validation을 통과해야 합니다.

  2. 2 영문 소문자만 사용 할 것

    모든 속성은 영문 소문자로만 작성합니다.

  3. 3 따옴표를 사용 할 때는 다음의 규칙을 준수 할 것

    - 작은 따옴표(' ')
    공백이 포함된 폰트명
    한글 폰트명
    문자열 데이터 타입
    filter 속성의 파라미터값
    - 큰 따옴표(" ")
    @charset 선언 시에는 속성
    - 따옴표 사용 금지
    url() 형태의 데이터 타입

  4. 4 들여쓰기는 다음과 같은 규칙을 준수 할 것

    HTML 요소의 들여쓰기 구조와 같이, 상위 block 엘리먼트의 자식들은 각각 1탭 들여씁니다.
    1탭의 크기는 공백 4칸으로 규정합니다.

  5. 5 W3C Validation 을 통과 할 수 있는 방식으로만 작성 할 것

    CSS는 사용 가능한 Hack과 CSS3 속성을 제외하고 W3C Validation을 통과해야 합니다.

  6. 6 빈 줄을 삽입 할 경우, 다음과 같은 규칙을 준수합니다.

    의미 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 넣을 수 있습니다.
    단, 빈 줄의 간격은 1줄을 초과하지 않도록 합니다.
    선택자와 중괄호”{}”가 중첩되는 Media Query 와 같은 케이스는 빈 줄을 반드시 삽입합니다.
