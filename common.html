1. HTML 코드 작성 규칙

HTML 코드를 작성하는 Mark up 을 진행 할 때 다음과 같은 기본 규칙을 준수합니다.

  1. 1 W3C Validation 을 통과 할 수 있는 구조로 작성 할 것

    HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation 을 통과해야 합니다.
    단, HTML5 DTD 선언 시 다음 오류 내용은 허용 할 수 있습니다.

    • iframe 의 frameborder, marginwidth, marginheight, scrolling 애트리뷰트

  2. 2 tag 작성시에는 영문 소문자만을 사용 할 것

    DTD 문서 명세를 제외한 모든 요소와 attribute는 소문자로 작성해야 합니다.

  3. 3 tag 내부의 attribute 와 class, id 등의 표기는 쌍따옴표("") 를 사용 할 것

    attribute, class, id 를 비롯한 tag 내부의 모든 속성 값은 쌍따옴표만을 사용 합니다.

  4. 4 tag 내부에 들어가는 특수문자는 웹 표준을 준수하여 코드로 작성 할 것

    tag 내부의 모든 특수문자는 HTML 5의 Character references 에 기재되어 있는 코드 값으로 표기 합니다.

    • HTML5 Character Rreferences Link

  5. 5 inline 선언은 가급적 지양 할 것

    tag 내부에 inline 방식으로 사용하는 css 속성 및 스크립트는 지양해야 합니다.

  6. 6 들여쓰기 규칙은 다음의 규칙을 준수 할 것

    들여쓰기를 하면 코드의 가독성이 높아지고 전체 HTML 구조를 쉽게 파악 할 수 있어 전체적인 코드의 재생산성을 높여줍니다.
    BEEKK 에서는 마크업의 중첩이 깊어질 때마다 자식 요소는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 규정합니다.
    문서 내에서는 반드시 탭을 이용하여 들여쓰기를 하며, 탭을 대신하여 공백으로 띄어 들여쓰지 않도록 유의합니다.

  7. 7 빈 줄을 사용하려면 다음의 규칙을 준수 할 것

    의미 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 만드는 것을 허용합니다.
    다만, 빈 줄은 1줄을 초과하지 않도록 유의합니다.

  8. 8 하나의 태그에 여러개의 속성을 기술해야 할 경우, 다음과 같은 순번을 지킬 것

    1. 1. 해당 태그를 사용시, 반드시 작성해야 하는 필수 속성 값을 먼저 작성합니다.
          단, required, readonly 와 같이 속성 그 자체가 값을 의미하는 경우
          모든 속성의 제일 마지막에 기술 할 것 을 명심하세요!
      • a 태그 : href → target
      • img 태그 : src → alt
      • label 태그 : for
      • input 태그 : type → name → value → ~ the others ~ → required → readonly
      example
      • - input [type="text"] 태그의 속성 기입 순서 :
            type → name → value → minlength → maxlength → placeholder
            → class → id → required → readonly
      • - input [type="checkbox"] 태그의 속성 기입 순서 :
            type → name → value → class → id → checked → required → readonly
      • - input [type="radio"] 태그의 속성 기입 순서 :
            type → name → value → class → id → checked → readonly

maxlength="16"
placeholder="아이디를 입력해주세요"

    2. 2. class
      • class 는 명사로 된 단어를 선택자로 사용 해야 하며, 합성어의 경우 각 단어간 연결을 언더바('_') 로 처리합니다.
      • 특히, BEEKK 의 CSS 컨벤션은 BEM STYLE GUIDE 를 지향하지 않습니다.
      • 과도한 class 사용은 html 코드 자체의 가독성을 떨어트리고 협업 능률을 저하시키게 되기 때문입니다.
      • 따라서, class 의 사용은 적재적소에 적절하게 쓰여야 합니다.
      • 모든 요소에 무의미하게 작성하는 것은 반드시 지양해야 합니다.

                </a>

    </li>
    <li>
        <button class="top_btn" onclick="scrollToTop()" title="최상단으로" >최상단으로</button>
    </li>
    </ul>
    3. 3. id
      • id 는 하이픈('-'), 혹은 언더바('_') 를 사용하지 않고 카멜케이스 규칙을 사용하여 기입해야 합니다.
      • id 역시 선택자이기 때문에, 기본적으로 명사 어구를 사용하여 기입하는 것을 권장합니다.
      • 여러 명사가 더해져 만들어진 합성어일 경우, 덧붙여진 명사의 알파벳 첫 글자를 대문자로 기입합니다.
      example

        <button class="arrow_btn only_mobile" onclick="promotionTopModalHandler()">닫기</button>
    </div>
    4. 4. javascript event
      • HTML 의 모든 속성을 전부 기술 한 뒤, 마지막에 javascript 와 관련된 event 항목들을 기입합니다.
      • 속성과 javascript event listner 가 뒤섞여 있을 경우 업무 효율이 현저히 떨어지고, 코드 리딩의 난이도가 급격히 올라가기 때문입니다.
      • 하나의 요소에 여러 javascript 이벤트를 중첩 바인딩 하는 경우에는 신중을 기해주세요!

2. CSS 코드 작성 규칙

CSS 코드를 작성하는 Mark up 을 진행 할 때 다음과 같은 기본 규칙을 준수합니다.

  1. 1 W3C Validation 을 통과 할 수 있는 방식으로만 작성 할 것

    CSS는 사용 가능한 Hack과 CSS3 속성을 제외하고 W3C Validation을 통과해야 합니다.

  2. 2 영문 소문자만 사용 할 것

    모든 속성은 영문 소문자로만 작성합니다.

  3. 3 따옴표를 사용 할 때는 다음의 규칙을 준수 할 것

    - 작은 따옴표(' ')
    공백이 포함된 폰트명
    한글 폰트명
    문자열 데이터 타입
    filter 속성의 파라미터값
    - 큰 따옴표(" ")
    @charset 선언 시에는 속성
    - 따옴표 사용 금지
    url() 형태의 데이터 타입

  4. 4 들여쓰기는 다음과 같은 규칙을 준수 할 것

    HTML 요소의 들여쓰기 구조와 같이, 상위 block 엘리먼트의 자식들은 각각 1탭 들여씁니다.
    1탭의 크기는 공백 4칸으로 규정합니다.

  5. 5 W3C Validation 을 통과 할 수 있는 방식으로만 작성 할 것

    CSS는 사용 가능한 Hack과 CSS3 속성을 제외하고 W3C Validation을 통과해야 합니다.

  6. 6 빈 줄을 삽입 할 경우, 다음과 같은 규칙을 준수 할 것

    의미 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 넣을 수 있습니다.
    단, 빈 줄의 간격은 1줄을 초과하지 않도록 합니다.
    선택자와 중괄호 “{ }“가 중첩되는 Media Query 와 같은 케이스는 빈 줄을 반드시 삽입합니다.

  7. 7 class 의 중첩 기술은 최대 5회를 넘어가지 않도록 할 것

    명확한 스타일 정의를 위하여 상위 클래스를 기술하는 습관은
    기본적으로는 매우 좋은 습관입니다.
    하지만, 너무 많은 중첩 클래스의 기술은 가독성을 심하게 떨어뜨리게 됩니다.
    5회 이상의 중첩 기술을 요할 경우에는 해당 요소에 새로운 class 를 부여하여
    독립된 개체로 기술하는 것을 권장합니다.

  8. 8 class 의 중첩 기술은 최대 5회를 넘어가지 않도록 할 것

    명확한 스타일 정의를 위하여 상위 클래스를 기술하는 습관은
    기본적으로는 매우 좋은 습관입니다.
    하지만, 너무 많은 중첩 클래스의 기술은 가독성을 심하게 떨어뜨리게 됩니다.
    5회 이상의 중첩 기술을 요할 경우에는 해당 요소에 새로운 class 를 부여하여
    독립된 개체로 기술하는 것을 권장합니다.

3. 작업 규칙 및 약속어 목록

  1. 1 색상 관련

    • BEEKK 에서 자주 사용되는 색상들은 /assets/css/common/color.css 에 코드형 변수로 기술되어 있습니다.
    • 필요시, 적절하게 응용하여 사용하면 작업 효율을 올릴 수 있습니다.
    • 다만, 직관적으로 인지하기 어려운 변수명이기 때문에 추후 리팩토링시 변경이 될 수 있습니다.
    example

  2. 2 반응형 관련

    1. 1. BEEKK 의 사이트는 ADMIN 관리자 단을 제외하고 모두 반응형 웹으로 되어있습니다.
      • BEEKK : 쇼핑몰 플랫폼
      • BEEKKERS : SNS 플랫폼
      • LIVE : 양방향 LIVE 방송 플랫폼
    2. 2. 컨텐츠의 범위는 최대 1200px, 최소 320px 입니다.
         지정어 .inner 를 사용하면 레이아웃을 잡는 시간을 줄일 수 있습니다.
      example
    3. 3. 공통 컴포넌트들 중 레이아웃 요소들의 미디어 쿼리는
         /assets/css/common/responsive.css 에 기술되어 있습니다.
         공통 컴포넌트 중 레이아웃 요소들은 다음과 같습니다.
      • header : 공통 헤더
      • gnb : 최상단에 위치한 메뉴
      • snb : 오른쪽 사이드와 모바일 하단 메뉴
      • lnb : 최상단 바로 아래에 위치한 메뉴
      • lnb_modal : 모바일에서 보이는 lnb 레이어 모달
      • container : 컨텐츠를 담는 그릇
      • footer : 회사 정보가 담긴 최하단 영역
      • modal : 일반적으로 재사용되는 공통 모달의 모바일 버전
      • detail_filter_wrap : 상품 상세정보 필터의 모바일 버전
      • newcoordi_filter_wrap : 뉴코디 메뉴에서 사용되는 상품 상세정보 필터의 모바일 버전
      • toast_alarm : 상품을 장바구니에 담거나, 관심 메뉴로 지정할 경우 잠시 보였다가 사라지는 토스트 알림
      • recommand_brand : 추천 브랜드 리스트
      example

         /assets/css/common/components.css 파일 하단에 모바일 디자인을 위한 미디어 쿼리가
         기술되어 있습니다.

    5. 5. 각 페이지별 미디어 쿼리는 pc 버전 css 가 기술된 파일 하단에
         components.css 와 동일한 방식으로 미디어 쿼리가 기술되어 있습니다.

// 드래그
function drag(){
    const drag = document.querySelector('.drag');
    let isDown = false;
    let startX;
    let scrollLeft;

    drag.addEventListener('mousedown', (e) => {
    isDown = true;
    startX = e.pageX - drag.offsetLeft;
    scrollLeft = drag.scrollLeft;
    });
    drag.addEventListener('mouseleave', () => {
    isDown = false;
    });
    drag.addEventListener('mouseup', () => {
    isDown = false;
    });
    drag.addEventListener('mousemove', (e) => {
    if(!isDown) return;
    e.preventDefault();
    const x = e.pageX - drag.offsetLeft;
    const walk = (x - startX) * 1; //scroll-fast
    drag.scrollLeft = scrollLeft - walk;
    });
}

// input custom select 함수
function customSelect(){
    var x, i, j, l, ll, selElmnt, a, b, c;

    x = document.getElementsByClassName("custom_select");
    l = x.length;
    for (i = 0; i < l; i++) {
    selElmnt = x[i].getElementsByTagName("select")[0];
    ll = selElmnt.length;

    a = document.createElement("div");
    a.setAttribute("class", "select_selected");
    a.innerHTML = selElmnt.options[selElmnt.selectedIndex].innerHTML;
    x[i].appendChild(a);

    b = document.createElement("div");
    b.setAttribute("class", "select_items select_hide");
    
    for (j = 1; j < ll; j++) {
        c = document.createElement("div");
        c.innerHTML = selElmnt.options[j].innerHTML;
    
        c.addEventListener("click", function(e) {
        var y, i, k, s, h, sl, yl;

        s = this.parentNode.parentNode.getElementsByTagName("select")[0];
        sl = s.length;
        h = this.parentNode.previousSibling;
    
        for (i = 0; i < sl; i++) {
            if (s.options[i].innerHTML == this.innerHTML) {
            s.selectedIndex = i;
            h.innerHTML = this.innerHTML;
            y = this.parentNode.getElementsByClassName("same-as-selected");
            yl = y.length;
            
            for (k = 0; k < yl; k++) {
                y[k].removeAttribute("class");
            }
            
            this.setAttribute("class", "same-as-selected");
            break;
            }
        }
        
        h.click();
        });

        b.appendChild(c);
    }

    x[i].appendChild(b);

    a.addEventListener("click", function(e) {
        e.stopPropagation();
        closeAllSelect(this);
        this.nextSibling.classList.toggle("select_hide");
        this.classList.toggle("select_arrow_active");
    });
    }

    function closeAllSelect(elmnt) {
    /*a function that will close all select boxes in the document,
    except the current select box:*/
    var x, y, i, xl, yl, arrNo = [];
    x = document.getElementsByClassName("select_items");
    y = document.getElementsByClassName("select_selected");
    xl = x.length;
    yl = y.length;
    for (i = 0; i < yl; i++) {
        if (elmnt == y[i]) {
        arrNo.push(i)
        } else {
        y[i].classList.remove("select_arrow_active");
        }
    }
    for (i = 0; i < xl; i++) {
        if (arrNo.indexOf(i)) {
        x[i].classList.add("select_hide");
        }
    }
    }

    document.addEventListener("click", closeAllSelect);
}
